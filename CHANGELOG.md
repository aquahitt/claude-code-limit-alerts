# Changelog

Формат — [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/), версии — [SemVer](https://semver.org/lang/ru/).

## [0.3.1] - 2026-07-23

### Fixed
- `refresh_via_cli()` в `usage-monitor.sh` теперь сверяет `cachedUsageUtilization.fetchedAtMs` до и после вызова `claude -p /usage`, а не доверяет одному коду возврата — при недоступности API (например, Team-подписка, заблокированная за неоплату) CLI мог завершаться успешно, но не обновлять кэш, из-за чего `usage-monitor.log` неделями писал ложное «refreshed local usage cache», маскируя реальную причину задержки в обнаружении сброса лимита.

## [0.3.0] - 2026-07-18

### Added
- `install.sh`/`update.sh` теперь поддерживают флаг `--proxy <url>` для фонового launchd-агента: без флага прокси-настройки (`HTTP_PROXY`/`HTTPS_PROXY`/`ALL_PROXY`/`NO_PROXY`, без учёта регистра) подхватываются автоматически из текущего шелла. Решает ситуацию, когда фоновый агент за корпоративным прокси/VPN не мог получить данные о лимитах — притом что `claude -p "/usage"` в обычном терминале работал нормально.
- `--proxy ""` отключает проброс прокси для агента.

## [0.2.2] - 2026-07-17

### Added
- `usage-monitor.log` теперь автоматически обрезается: как только файл превышает 1МБ, остаются последние 2000 строк, чтобы лог не рос бесконечно на фоновом cron.

## [0.2.1] - 2026-07-17

### Fixed
- `claude -p "/usage"` в `refresh_via_cli()` теперь резолвится явными кандидатами (`~/.local/bin/claude`, Homebrew, `/usr/local/bin`), а не только через `PATH` — launchd запускает `cron`-джобу с голым `PATH` (`/usr/bin:/bin:/usr/sbin:/sbin`, без профиля пользователя), из-за чего фикс 0.2.0 для team/протухших токенов в реальном фоне не срабатывал (`command -v claude` ничего не находил).

## [0.2.0] - 2026-07-17

### Added
- Версионность проекта (`VERSION`, этот `CHANGELOG.md`).
- `update.sh` — синхронизирует установленные скрипты, launchd-агент и хуки с текущим состоянием локальной копии репозитория, без `git pull` внутри себя. Поддерживает `--dry-run`.
- `usage-monitor.sh status` теперь первой строкой печатает установленную версию.

### Changed
- Логика регистрации хуков (`add_hook` и обёртки над ней) вынесена в `lib/hooks.sh`, используется и `install.sh`, и `update.sh` — раньше была продублирована.

## [0.1.0] - 2026-07-16

### Added
- Мониторинг лимитов Claude Code (сессия 5ч, неделя): уведомления и сообщения в интерфейсе Claude Code при приближении к порогам 80%/95% и при сбросе окна.
- Уведомления «требуется внимание» (баннер + звук) на события Notification/Stop.
- Statusline-обёртка с процентами всех лимитов.
- Фоновый launchd-агент (проверка каждые 5 минут), `install.sh`/`uninstall.sh`.

### Fixed
- Ложные алерты о сбросе окна из-за ±1с джиттера `resets_at` в ответе API.
- Тихие сбои опроса (`fetch_usage`) теперь пишутся в `usage-monitor.log` с указанием причины, вместо молчаливого `exit 0`.
- Для team/org-аккаунтов (403 на живом эндпойнте) и протухших OAuth-токенов — принудительное обновление локального кэша через `claude -p "/usage"` в режимах `cron`/`status`.
