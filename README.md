# claude-code-limit-alerts

**Русский** | [English](README.en.md)

Уведомления для Claude Code: мониторинг лимитов подписки (приближение к исчерпанию,
сброс окна, проценты в statusline) и «требуется внимание» — баннер со звуком, когда
Claude ждёт разрешения, ответа или закончил задачу.

> 🍎 **Пока только macOS.** Поддержка Windows и Linux запланирована — см. [Roadmap](#roadmap).

## Что вы получаете

**macOS-уведомления и сообщения в интерфейсе Claude Code:**

```
🟡 Лимит «Сессия (5ч)»: 82%, сброс в 18:00
🔴 Лимит «Неделя (все модели)»: 96% — почти исчерпан, сброс в 10:00
♻️ Лимит «Сессия (5ч)» сброшен (было 96%, сейчас 3%)
```

**Statusline с процентами всех лимитов** (зелёный / жёлтый ≥ 80% / красный ≥ 95%):

```
[ваш statusline] | 5ч 71% · 7д 7% · Fable 4%
```

**Уведомления «требуется внимание»** — когда Claude ждёт разрешения/ответа или
закончил работу, приходит баннер со звуком. В заголовке — имя сессии (или папка
проекта) и git-ветка, в подзаголовке — последний ваш запрос, чтобы сразу понять,
в какой терминал переключиться:

```
Claude: stroi-homes (develop)          ← какая сессия
исправь баг с уведомлениями…           ← над чем работает
Нужно разрешение или ответ             ← что случилось   (звук: Funk)
Задача завершена                       ←                  (звук: Glass)
```

Звук проигрывается напрямую через `afplay` — работает, даже если баннеры
не разрешены в Центре уведомлений.

**Ручная проверка в терминале:**

```
$ ~/.claude/scripts/usage-monitor.sh status
Сессия (5ч)           71%  сброс: 16.07 18:00
Неделя (все модели)    7%  сброс: 17.07 10:00
Неделя (Fable)         4%  сброс: 17.07 10:00
```

## Как это работает

- Скрипт опрашивает тот же endpoint, что и команда `/usage` в Claude Code
  (`api.anthropic.com/api/oauth/usage`), OAuth-токен берётся из macOS Keychain —
  никаких дополнительных ключей и логинов не нужно.
- **Хуки Claude Code** (`Stop` + `SessionStart`) показывают предупреждения прямо
  в интерфейсе после хода и при старте сессии.
- **launchd-агент** проверяет лимиты каждые 5 минут в фоне — уведомление о сбросе
  окна придёт, даже если Claude Code закрыт.
- **Statusline-обёртка** дописывает проценты к вашему существующему statusline
  (он сохраняется и продолжает работать) — данные берутся из локального кэша,
  без сетевых запросов.
- Уведомления не спамят: по одному на порог (80% и 95%) в рамках одного окна,
  сброс отмечается только если использование было ≥ 50%.

Подробности — в [docs/how-it-works.md](docs/how-it-works.md).

## Установка

Требования: macOS, [jq](https://jqlang.github.io/jq/) (`brew install jq`),
Claude Code с авторизацией по подписке (Pro/Max).

```bash
git clone https://github.com/aquahitt/claude-code-limit-alerts.git
cd claude-code-limit-alerts
./install.sh
```

Флаги установщика:

| Флаг | Действие |
|---|---|
| `--no-statusline` | не трогать statusline |
| `--no-launchd` | без фонового агента (только хуки) |
| `--no-attention` | без уведомлений «требуется внимание» |
| `--lang en` | уведомления на английском (по умолчанию — русский) |

После установки перезапустите Claude Code (или откройте один раз `/hooks`),
чтобы подхватились новые хуки. Перед изменением `~/.claude/settings.json`
создаётся бэкап.

## Обновление

```bash
git pull
./update.sh
```

Сверяет установленную версию (`~/.claude/scripts/.limit-alerts-version`) с
`VERSION` в репозитории, перекладывает изменившиеся скрипты, перезагружает
launchd-агент при изменении конфигурации и дозаявляет хуки — без
переустановки того, что вы не ставили (`--no-statusline`/`--no-launchd`/
`--no-attention` по-прежнему уважаются). Флаг `--dry-run` показывает, что
будет сделано, ничего не меняя. Список изменений между версиями — в
[CHANGELOG.md](CHANGELOG.md).

## Настройка

Пороги и поведение настраиваются переменными окружения (или правкой дефолтов
в начале `usage-monitor.sh`):

| Переменная | По умолчанию | Что делает |
|---|---|---|
| `UM_WARN` | `80` | порог 🟡-предупреждения, % |
| `UM_CRIT` | `95` | порог 🔴-предупреждения, % |
| `UM_RESET_MIN` | `50` | минимальное использование, при котором сброс окна заслуживает уведомления |
| `UM_CACHE_TTL` | `60` | срок жизни кэша ответа API, сек |
| `UM_LANG` | `ru` | язык сообщений: `ru` или `en` |

Интервал фоновой проверки — `StartInterval` (секунды) в
`~/Library/LaunchAgents/com.claude.usage-monitor.plist`.

## Удаление

```bash
./uninstall.sh
```

Выгружает launchd-агент, убирает хуки из настроек, возвращает прежний statusline
и удаляет скрипты.

## FAQ

**Уведомления не приходят.**
Проверьте, что уведомления от «Script Editor» разрешены:
Настройки macOS → Уведомления. Скрипт шлёт их через `osascript`.

**Это официальный инструмент Anthropic?**
Нет. Используется недокументированный endpoint, которым пользуется сам Claude Code
для команды `/usage` — формат ответа может измениться. Скрипты в этом случае
тихо перестанут показывать данные (ничего не сломается).

**Токен протухнет?**
Claude Code сам обновляет OAuth-токен при работе; скрипт всегда читает актуальный
из Keychain. Если токен невалиден, проверка тихо пропускается до следующего цикла.
